# AI Floor Plan Editor & Generator

Интерактивное веб-приложение для автоматической разметки чертежей и генерации планировок квартир с использованием искусственного интеллекта. Проект объединяет детекцию объектов (**YOLOv8**), генерацию изображений (**U-Net / GAN**) и удобный графический веб-интерфейс на **HTML5 Canvas**.

---

##  Ключевые возможности

*   **Автоматическая разметка планировок (AI Detection):** Локализация стен, окон и дверей на загруженном чертеже с помощью обученной модели сегментации **YOLOv8**.
*   **Интерактивный редактор (JS Canvas):** Возможность вручную корректировать, перемещать и создавать новые полигоны разметки (стены, двери, окна) прямо на холсте в браузере.
*   **Генерация планировок (AI Generation):** Процедурная или нейросетевая (**U-Net/GAN**) генерация новых планировок на основе заданной площади и формы помещения.
*   **Экспорт разметки:** Быстрое сохранение готового векторного датасета в профессиональном формате **COCO JSON** для дальнейшего обучения моделей.

 Архитектура и технологии
Бэкенд (Python & Flask)
Flask — легковесный веб-сервер, выступающий в роли REST API. Он обрабатывает загрузку картинок, координирует работу нейросетей и отдает результаты фронтенду.

PyTorch & Ultralytics — запуск глубоких нейросетей на CPU или GPU (CUDA).

Нейросетевое ядро (AI Core)
Сегментация и детекция (core/detector.py):
Использует модель сегментации YOLOv8 (загружает веса models/best_seg.pt). Если модель сегментационная, она возвращает точные полигоны стен, окон и дверей. При использовании обычной модели детекции детектор автоматически достраивает прямоугольные рамки (Bounding Boxes) до 4-точечных полигонов, гарантируя стабильность фронтенда.

Генерация планировок (core/generator.py):
Построен по гибридному принципу. При наличии весов generator.pt активируется модель U-Net, обученная в режиме cGAN (Pix2Pix) для перевода контуров в цветные планировки. Если весов нет, активируется процедурный генератор на базе библиотеки Pillow (PIL), который программно строит планировки нужной площади и формы.

Фронтенд (HTML5 Canvas & Vanilla JS)
Отображает оригинальный чертеж и накладывает на него интерактивные векторные полигоны.

Позволяет пользователю редактировать вершины масок, переназначать классы элементов (wall, window, door) или рисовать новые зоны с нуля.

Запуск программы
1. Клонирование репозитория и переход в папку проекта
```text
git clone <url_вашего_репозитория>

cd gemeni
```
2. Создание и активация виртуального окружения
Windows:

```text
python -m venv .venv
.venv\Scripts\activate
```
3. Установка зависимостей
```
pip install -r requirements.txt
```
4. Подготовка весов моделей
Убедитесь, что в папке models/ находятся файлы весов:

Обученная модель детектора: models/best_seg.pt

(Опционально) Модель генератора: models/generator.pt (при отсутствии будет работать процедурный генератор-заглушка).

5. Запуск веб-приложения
```
python app.py
```
После запуска откройте в браузере страницу: http://127.0.0.1:5000/

 Обучение моделей
Обучение YOLOv8
Данные для обучения разметчика должны быть подготовлены в формате YOLO Segmentation внутри папки floor_plan_training/. Настройки классов задаются в floor_plan_training/data.yaml.

Чтобы запустить обучение модели:

```
python train.py
```
Результаты обучения, графики метрик и веса будут автоматически сохранены в директорию runs/.

 Формат экспорта данных (COCO JSON)
Сформированный на интерактивном холсте датасет экспортируется в стандартный формат аннотаций COCO, что делает его полностью совместимым со сторонними библиотеками (например, Detectron2 или MMDetection):
```

  ],
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "category_id": 1,
      "segmentation": [[120.5, 300.2, 150.1, 300.2, 150.1, 450.8, 120.5, 450.8]],
      "area": 4518.0,
      "bbox": [120.5, 300.2, 29.6, 150.6],
      "iscrowd": 0

  "categories": [
    { "id": 1, "name": "wall" },
```
